Supprimer les doubles vérifications

Actuellement, l'IA appelle parfois check_room_availability 2 fois pour le même email (ex : Rebecca — 2 appels). Le 2ème appel filtre par type de chambre, mais c'est souvent redondant car les infos sont déjà dans le 1er résultat.

Économie estimée : -30% de tokens sur les emails complexes = ~0,05 €/email au lieu de 0,08 €
💾

Mettre en cache les données statiques

Les détails des chambres, la liste des restaurants, les plages — ces données ne changent quasiment jamais. Au lieu de les re-chercher à chaque email, on peut les garder en mémoire et les injecter directement dans le prompt.

Économie estimée : -1 à 3 appels d'outils par email = réponses plus rapides + moins de tokens
📏

Optimiser le prompt système

Le prompt système actuel fait ~2000 tokens. Il inclut toutes les règles, le contexte client, et l'historique de conversation. On peut le compresser en gardant l'essentiel — chaque token en moins dans le prompt = économie sur chaque email.

Économie estimée : -20% de tokens d'entrée = ~0,01-0,02 € économisés par email
